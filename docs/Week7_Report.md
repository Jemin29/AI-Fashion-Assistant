The primary objective of Week 7 was to transition the AI Fashion Assistant prototype into a secure, hardened, and production-ready enterprise REST API service. * Integrated the `Falconsai/nsfw_image_detection` model as a FastAPI middleware filter to automatically scan incoming/outgoing multipart file streams and reject unsafe content.
* Sandboxed execution paths to prevent server crashes on malformed files.

### 4. Digital Copyright Protection (Watermarking)
* Designed a transparent, non-obtrusive text watermark (`"Generated by AI Fashion Assistant"`) applied with drop-shadow styling to the bottom-right corner of all generated PNG/JPEG images.
* Integrated watermark rendering as a post-processing filter inside the image generation service pipeline.

### 5. Multi-Layer Monitoring & Service Health Diagnostics
* Developed an aggregated diagnostic gateway (`/health`) querying the individual operational states of RAG, ControlNet, LoRA, Redis, and Generation services.
* Added resource monitoring foundations to track memory, CPU, and worker task queue lengths.

## Key Challenges & Resolutions

### 1. RAG Router Intent Mismatch
* **Problem**: The RAG assistant routes requests asking about fabric characteristics to an `explain_fabric` intent, which returned data under the `response` key instead of the `answer` key expected by tests and the API spec.
* **Resolution**: Intercepted the returned dictionary in the `/chat` route of `routers/rag.py` and standardized it by mapping the `response` key to the `answer` key if missing.

### 2. Recommendation Service Signature Mismatch
* **Problem**: The underlying Gradio-focused recommendation service required specific unpacked keyword arguments, whereas the week 5/7 routers passed a single `preferences` or `profile` dictionary payload, leading to `TypeError` exceptions.
* **Resolution**: Refactored the `RecommendationService` wrapper to safely unpack payload dictionaries into their individual fields (`gender`, `style`, `occasion`, `fit`, `n`, `color`) before invoking the core engine.

### 3. Path Imports & Missing Package References
* **Problem**: A `NameError: name 'Path' is not defined` error was triggered in `routers/controlnet.py` when attempting to verify and create the target sketches directory.
* **Resolution**: Added `from pathlib import Path` to the router's imports list.

### 4. Dict Serialization & Health Checks
* **Problem**: Direct dict returns from week 6 mock services caused `AttributeError: 'dict' object has no attribute 'data'` crashes on health check endpoints expecting `ServiceResult` objects.
* **Resolution**: Implemented dynamic type checking in the service layer using `hasattr(res, "data")` to safely extract results.
